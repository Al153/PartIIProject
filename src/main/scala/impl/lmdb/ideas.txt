Ideas for the LMDB implementation


Addressing Objects:
Key path.

Need to store in database:


    Index tables: One per column per Object type:
        looked up with:
            db:index:$objectType:$Column:$commit:$value  -> LazySet(objectId)

    emptyIndex:
        db:emptyIndex:$objectType:$commit -> LazySet(objectId)


    objectsAndRelations:
        db:outgoingRelations:$objectType:$id:$commit -> LazySet(relationId, objectType, objid)
        // reverse relation lookup
        db:incomingRelations:$objectType:$id:$commit -> LazySet(relationId, objectType, objid)

    objectRetrieval:
        For picking objects out of the database, one per object
        db:objects:$objectType:$id -> DBObject


ControlTables:
    CommitsCounter
        db:nextCommit -> commit //  mutable
    
    ViewsCounter
        db:nextView -> view // mutable

    ObjectsCounter
        db:nextObject -> View // mutable
    
    AvailableViews
        db:views -> LazySet(view) // mutable
    
    DefaultView
        db:default -> view // mutable
    
    ViewsTable
        db:ViewLookup:$ViewId -> LazySet(commit)
